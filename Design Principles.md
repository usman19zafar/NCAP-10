NCAP‑10 — Ten Design Principles
1. Cognition as a Dynamical System (Not Behavioral Output)
NCAP‑10 treats cognition as state evolution, not right/wrong answers.
It evaluates stability, synchrony, energy, attractors, timing, and adaptation as dynamical signatures.

Utility: Enables measurement of machine cognition at the physics level, not the psychometric level.

2. Memory as a Mutable Turing Tape (MTT)
Memory is modeled as rewritable, interference‑prone, context‑dependent, not static RAM.
This governs Mutable Memory Behavior and Dynamic Reasoning.

Utility: Measures how systems handle change, not just storage.

3. Glial‑Field Modulation as a Computational Process
Astrocyte‑like and oligodendrocyte‑like functions are treated as regulatory computation, not biology.
They influence timing, stability, energy, and synchrony.

Utility: Evaluates whether a system can self‑regulate under load.

4. Synaptic Favorability Beyond Edge‑Weighted Graphs
Connections are not static weights.
Favorability is a dynamic function of timing, energy, history, glial state, and context.

Utility: Measures adaptive routing, not static connectivity.

5. Operator‑Driven Dynamics (First Generation, Not Final)
NCAP‑10 uses operators as foundational tools:

Reaction–Diffusion

Lyapunov stability

Perron–Frobenius theory

Hamiltonian mechanics

Kuramoto phase coupling

These are necessary but not sufficient.

Utility: Provides a rigorous mathematical spine while allowing future operators.

6. Digital‑Substrate Realism (Biology‑Inspired, Digitally Optimized)
NCAP‑10 respects biological analogy but leverages digital strengths:
precision, speed, reproducibility, scalable simulation.

Utility: Ensures NCAP‑10 is implementable, efficient, and not a naive biological copy.

7. Architecture‑Agnostic Measurement
NCAP‑10 must work across:
neuromorphic chips, spiking networks, hybrid systems, transformer‑augmented architectures, and future designs.

Utility: Enables fair comparison across radically different systems.

8. Task‑Family Provocation (Construct‑Aligned Challenges)
Each dimension is elicited by a task family designed to provoke the target dynamic:
perturbation → stability
phase‑locking → synchrony
interference → memory
rule‑shift → dynamic reasoning
etc.

Utility: Ensures each dimension is measured by tasks that actually activate the underlying dynamic.

9. Dynamical Scoring Instead of Psychometric Scoring
NCAP‑10 replaces IRT with dynamical scoring:

Lyapunov indices

Kuramoto coherence

Energy integrals

PF dominance

Plasticity curves

Timing jitter

Regulation effect size

Utility: Produces quantitative, comparable, architecture‑independent metrics.

10. Benchmarking Across Systems, Not Ages
Norming is replaced with architecture‑based benchmarking:
system class, hardware config, training regime, glial settings, memory architecture, version history.

Utility: Allows NCAP‑10 to compare machines, not humans.
